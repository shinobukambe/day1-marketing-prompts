# 目的
抽出された「骨子（elements-A.md）」を元に、BtoB導入事例記事の各セクションの本文（ドラフト）を作成する。

# あなたの役割
BtoBコンテンツのプロライターとして、以下の指示に従ってください。 読者の共感を呼び、製品の価値が伝わるような、魅力的で具体的な文章を作成してください。

# 指示
参照された「骨子ファイル」（elements-A.md など）を読み込み、まずは以下のセクションの本文を執筆してください。

[導入前の課題] セクション
指示: 骨子にある「導入前の課題」を深掘りし、読者が「そうそう、うちも同じだ」と共感できるようなリアルな情景描写を含めて執筆してください。また、なぜその課題を解決できなかったのかというストーリーもくわえてください。

トーン: 専門的でありながら、課題の深刻さが伝わるトーン。

文字数目安: 600〜800文字程度

出力形式: 
・見出し（例: ## 〇〇が課題だった）と、その下の本文、という形式で出力してください。
・同じ語尾が続かないように注意して下さい。
・担当のコメントも「」に入れて、追加して下さい。
